# CI Audit Memo — Community Information (64 files)

## Coverage Statement

I surveyed **all 64 files** in `LIONSBERG Community Information/`. Of those, I read in full or substantially (>50% of file): the three index-level files (Meetings.md, Project Proposals.md, Writing Project Proposals.md), all three People/ files, and approximately 22 representative meeting notes spanning every series (Wiki Posse, Map Weavers, Meta Project Navigation, Social Architecture, Resourcing, Lionsberg Africa/Kenya, Currency and Measurement, Pull Planning, Jordan–Pete dyad, Jordan–Wendy, Jordan–A, Lionsberg 2023.x weekly meetings, Lionsberg 2022.11.x). ## Headline Finding

**This entire folder is a personal/historical archive of a specific 2022–2023 formation cycle.** It is not canon material. It documents the actual lived process by which Jordan and a small group (Pete Kaminski, Wendy McClean, Wendy Elford, Bill Larson, Judy Bentham, Forrist, Michael Grossman, Vincent, Jonathan Sand, Killu, Marianne, Donna Nelham, Trae, Erik, Jason Schulist, Elizabeth, Hank, Gil, George Orbelian, Matt, Estelle, Jacque, Moses, Patrick, plus Kenyan partners) **lived their way into** the patterns that have since become canonical Cards (Circle, Fellowship Meeting, Pull Planning, Field of Agreements, Federation, OmniSpection, Five-Phase Project Pattern, Talking Stick, Network of Commitments, etc.). The canonical articulation has already been extracted into the Pattern Language, Playbook, PDG, and Codex over the last three years. Almost nothing genuinely new remains to surface.

The folder's primary value is **historical**, not generative: it documents how the Seed was actually grown. The discipline here is to **leave it as personal history** and not contaminate canon with the relational specifics, names, dollar figures, and place-specific projects that fill these pages.

## Coherence Observations

1. **The whole folder should be reframed as "Lionsberg Origins — Meetings & Correspondence 2022–2023" or moved to a clearly-bracketed "Historical Archive" scope.** Its current placement in the root sidebar implies it is an ongoing live channel of community information, but it is in fact a closed historical chapter. New community information has not been added here in ~3 years. The naming gives a misleading signal to anyone arriving fresh.

2. **The "Meta Project" terminology** runs through almost every file as the dominant frame, but Meta Project was abandoned as a name by late 2022 in favor of LIONSBERG itself. Future readers will be confused. A brief framing header at the top of the Meetings folder ("These notes use 'Meta Project' as an earlier name for what became LIONSBERG") would help.

3. **The Project Proposals folder (2 files) should fold into the canonical Quest / Request For Guidance / RFG structure.** The "Writing Project Proposals" file with its Need / Context / Implications / Solution / Measurable Goal / Plan / Timeframe template is essentially an early articulation of the Quest Proposal pattern. The Card canon should reference it; the folder itself can be archived. The contributors named there (Bill Larson, Judith Benham, Jonathan Sand) get their credit through the historical archive, not through canonical attribution.

4. **The Meetings folder demonstrates an actual five-phase project pattern** (Vision/Intention → Analysis → Engagement → Strategy/Plan → Embody/Enact → Retrospection) more clearly than many of the PDG documents do. This deserves a careful read by whoever is curating the PDG Cards — the five-phase pattern is articulated cleanly in Project Proposals.md and could sharpen the Plan-Do-Check-Adjust / Five-Phase Cards.

5. **The Lionsberg Africa series is qualitatively different** from every other meeting series — it documents real boots-on-the-ground operational work (school, transportation, water, agriculture, training) in a specific geography with specific dollar figures and specific Christian devotional framing. It does not belong in the same conceptual bucket as the Wiki Posse or Map Weavers or Meta Project Navigation meetings, which were primarily abstract/architectural. Consider separating: `Meetings/Origins — Architectural Conversations 2022-2023/` and `Meetings/Origins — Kenya Operational Work 2022-2023/`. The latter is also more clearly Jordan's personal apostolic work and may belong in a different scope altogether (perhaps Jordan's personal archive).

6. **The People/ folder is structurally underdeveloped.** It has only three files (Jordan, Pete, Lionsberg Co-Creators) — yet ~25 named individuals appear repeatedly across the meetings. Either the People/ folder should be removed entirely (since the universal principle is that individuals stay personal and don't get profiled), or it should be much fuller and live elsewhere. **My recommendation: remove the People/ folder.** Jordan's profile already exists more substantively in other places (Above The Chaos, Jordan's Short Story, etc.); Pete's profile is his own sovereign self-presentation and could live on his own site or in a personal/contributor index. Lionsberg Co-Creators is a conceptual page about Fibonacci cycles, not really about people.

7. **The maturity-stage label "seed/sprout/seedling/sapling/adult"** mentioned in CLAUDE.md is not applied here. Every file in this folder should arguably carry a "historical archive" or "mature/sealed" tag so future curators understand they are not editable in the way live canon files are. The folder is a kind of seedbed-already-fruited.

8. **The folder is a goldmine for one thing not directly extractable as Cards: testimony.** Many of these meeting notes contain raw direct quotes from named individuals that, *with their consent*, could become powerful Testimonies in the Testimony genre. But this is testimony work for a separate phase, requiring explicit author consent. Do not extract testimony without consent.

9. **No NBSP / two-space-ending issues observed** in the sample I read — the folder appears to predate or be unaffected by the global sweep recorded in MEMORY.md. If a sweep is being done across the wiki, this folder is fine as is.

10. **The folder contains zero "LIØNSBERG" canonical-glyph usage** because it predates the 1.5 phase adoption. This is correct for historical artifacts; do not retroactively re-glyph the meeting notes. They should preserve their original "Lionsberg" spelling as the historical record.
