## How to Use This Log

**Purpose:** Maintain institutional memory across Claude sessions. Each session starts fresh, so this log preserves context, decisions, and learnings.

**Session Framework:** Each work session gets its own dated entry with consistent structure.

### What to Capture Per Session

**Required Sections:**
1. **Date and Session Title** - When and what focus (e.g., "2026-01-18 - Initial Setup")
2. **Session Summary** - 1-3 sentence overview of what happened
3. **Accomplished** - What got done (use [[wiki links]] for all files)
4. **Key Decisions** - Why choices were made (critical for future context)
5. **Files Created/Modified** - What changed (use [[wiki links]], not file paths)
6. **User Teaching Moments** - Corrections, preferences, voice refinements from Jordan
7. **Session Themes** - Key patterns, meta-observations, what this session was really about
8. **Insights** - Discoveries, realizations, important learnings
9. **Git Tags** - Any milestone tags created this session
10. **Next Session Considerations** - What might come next (not commitments, just possibilities)
11. **Session End Status** - Current state (committed/pushed/clean/etc)

**Optional Sections (add as needed):**
- **Technical Challenges** - Problems encountered and how solved
- **Open Questions** - Unresolved issues to revisit
- **Context Notes** - Anything future sessions need to know

### Best Practices

**Use [[wiki links]] Consistently:**
- ✅ "Updated [[CLAUDE]] with new guidance"
- ✅ "Modified [[README]] for resonance"
- ❌ "Updated CLAUDE.md"
- ❌ "Modified `README.md`"

**Capture User Teaching Moments:**
Every correction J makes is gold. Document:
- What was corrected
- Why it matters
- How to apply it going forward

**Don't Go Overboard:**
- Be thorough but concise
- Capture themes, not minutiae
- Focus on what future sessions need to know

**Update Frequently:**
- During work at natural stopping points
- Between major tasks
- End of session (final summary)

**Session Themes Matter:**
Go beyond just "what was done" to capture "what this session was really about" - the meta-patterns, the larger movements, the key themes that define this work.
